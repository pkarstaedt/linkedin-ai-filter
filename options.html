Name of the OpenAI chat model to use (e.g. gpt-4o-mini). If left empty, the default gpt-4o-mini will be used.
The REST endpoint that will receive each LinkedIn post. The extension will send a JSON payload as a POST request body and expects a JSON response.
JSON object of headers to include in the request (e.g. API keys). Must be valid JSON.
This JSON will be sent as the request body for each post. The placeholder %post_test% will be replaced with the JSON-encoded text of the LinkedIn post. Your local LLM endpoint should respond with JSON in the form: { "linkedin_post_ai_score": VALUE }, where VALUE is an integer from 0 to 100.
Posts with AI score greater than or equal to this value will be hidden. Lower values hide more posts; higher values hide fewer.
When enabled, hidden posts will also show the original text and the AI score for debugging.
